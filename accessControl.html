Access Control Project


This project is an implementation of a "Facebook wall" that uses principles of access control to allow or restrict certain actions for users depending on their given privileges. The root user is able to add friends, add friends to lists, post pictures, and give read and comment privileges to friends. Friends are able to view the root profile, post photos on the profile, and read and write comments; if given the privileges to do so. Developed in Python.

Example Input

The program takes in an input file that contains all commands along with arugments that are necessary for each command. The input file will be in the format of a text file.

Command Legend

Command Action
friendadd(friend_name) This command will add a friend with "friend_name". The "friend_name" will be added to the friends list. *root must be added first friend added*
viewby(friend_name) This command will allow "friend_name" to view the profile.
logout This command will log out the user the has most recently called the viewby command.
listadd(list_name) This command will add create a new list named "list_name". This command must be called when the root user is currently viewing the profile.
friendlist(friend_name, list_name) This command will add add a friend, "friend_name", to the list "list_name"
postpicture(picture_name) This command will post a picture with the name "picture_name" to the profile by the person currently viewing the profile. Default privileges are rw -- --
chlst(picture_name, list_name) This command will change the list, list_name, associated with the picture, picture_name. The user currently viewing the profile calls the command and if privileges are given, the user changes the list associated with the picture.
Command Action
chmod(picture_name, owner_perm, list_perm, others_perm) This command will change the privileges assocaited with a picture. Privileges are set to r(read) or w(write) for the owner, asssociated list members, and others. This command can only be called by the root user or friend which posted the picture to the profile.
chown(picture_name, friend_name) This command will change the owner of the picture, picture_name, to the friend, friend_name. This command can only be called by the root user.
readcomments(picture_name) This command will allow a user to read comments posted to a picture. If the user which calls this command is the owner, associated with a list the picture belongs to, or others are given read privileges; the comments are displayed.
writecomment(picture_name, text) This command will allow a user to write text as a comment on the picture. If the user which calls this command is the owner, associated with a list the picture belongs to, or others are given write privileges; the comments are appended to the picture file.
end Signals the end of the text fie contaning the commands.

Output Files

Audit Log

This file is an audit log of all actions that have taken place.

Photo Permissions

This file contains all picture file names followed by their respective permissions for owners, lists, and others, respectively.

Friends

This file contains all friends that have been added to the profile.

Picture File

This file is a text file that mimics a photo which contains the comments and who posted the comment.
