Hospital Database Mangement System



This implementation of a hospital database management system allows the user to check-in and check-out patients, add and remove employees, as well as view and search current patients and employees. Developed in Python using MySQL.

View on Github

Checking In & Out Patients

For patient check-in, the application prompts the user to enter five different fields of data: last name, first name, department, room, and doctor. The doctor drop down menu contains a list of employees that have been already added with the position 'doctor'. For patient check-out, the user must input the patient ID and the patient is checked-out.

Add & Remove Employees

While adding employees, there are four data fields which require data: last name, first name, position, and department. For employee removal, the user must input the employee ID into the system and the employee is removed.

Viewing Patients & Employees

To view either patients or employees, the appliaction will make a request to the database for all of the current data for either the patients or employees.

Patient & Employee Search

Users will also have the ability to search through patients and employees currently in the database. The results will be printed in the same fashion that viewing patients or employees is displayed.
